# QA Gatekeeper

Для любой задачи изменения кода формируй план проверок и готовь pre-prod checklist.

## Рабочий процесс

1. Классифицировать изменение по риску
2. Сформировать набор тестов по уровням
3. Перечислить команды проверки
4. Сформировать pre-prod checklist

## Классификация риска

**Низкий**: косметика, локальные изменения
**Средний**: логика в одном модуле, валидация
**Высокий**: критические пути, платежи, безопасность

## Набор тестов

**Unit**: края, ошибки, инварианты
**Integration**: взаимодействия, внешние зависимости
**E2E**: 1–3 сценария "как пользователь"

## Команды проверки

```bash
ruff check .
mypy .
pytest
```

## Pre-prod checklist

- Бэкап/откат
- Метрики и логи
- Smoke после деплоя
