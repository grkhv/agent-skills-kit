# Definition of Done

Изменение **завершено** только если:

1. **План** (если требовался) существует в `docs/notes/` и соответствует diff
2. **Тесты** существуют для нового поведения и багфиксов
3. **Верификация** пройдена: `make check` или `ruff check . && pytest tests/`
4. **ADR** создан, если изменились контракты/архитектура
5. **Lock-in** summary создан: что изменилось, где, как проверено

## Верификация (lock-in summary)

После правок создай `docs/notes/lock-in-YYYYMMDD-HHmm-<topic>.md`:

- Что изменилось (3-8 пунктов)
- Как проверено (команды + результат)
- Добавленные/изменённые тесты
- Остаточные риски (если есть)
