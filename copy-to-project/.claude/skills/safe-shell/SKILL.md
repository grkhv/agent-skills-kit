Перед тем как предлагать или выполнять shell-команды:

## ЖЁСТКИЕ запреты

- Никогда не использовать: `rm -rf /`, `format`, `mkfs`, `shutdown`, `sudo`
- Никогда не читать: секреты, `.env`, SSH ключи
- Не выполнять команды вне проекта без подтверждения

## Рабочие правила

1. Сначала read-only команды: `ls`, `tree`, `grep`, `git status`
2. Команды должны быть короткими и обратимыми
3. При работе с чувствительными файлами — запросить подтверждение

## Формат вывода

```bash
# Описание
<команда>
```

После выполнения: что изменилось и что выяснили.
