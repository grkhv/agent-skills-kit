Найди баги в GitHub Issues текущего репозитория и исправь их.

## Алгоритм

### 1. Определи репозиторий

- Прочитай `.git/config` → извлеки `url` из `[remote "origin"]`
- Определи `owner/repo` из URL
- Если remote не найден — спроси пользователя

### 2. Найди баги

- Открой `https://github.com/{owner}/{repo}/issues`
- Ищи issues с лейблом `bug`
- Составь список: номер, заголовок, описание

### 3. Для каждого бага

```
[ ] Шаг 1: Понять проблему (описание issue + воспроизведение)
[ ] Шаг 2: Найти связанный код (используй serena, grep, find_symbol, структура)
[ ] Шаг 3: Определить root cause
[ ] Шаг 4: Исправить
[ ] Шаг 5: Проверить (тесты, линтер)
[ ] Шаг 6: Сообщить результат
```

### 4. Отчёт

После исправления каждого бага выведи:

```markdown
## Bug #{номер}: {заголовок}
- **Причина**: ...
- **Исправление**: {файл}: {что изменено}
- **Проверка**: {команда} → {результат}
```

## Ограничения

- НЕ закрывай issues автоматически (нет доступа к API)
- НЕ создавай PR — только локальные изменения
- Если баг требует уточнений — сообщи пользователю
- Если баг не воспроизводим — пропусти и укажи причину

## Когда НЕ применять

- Баги в стороннем коде (зависимостях)
- Баги, требующие инфраструктурных изменений (CI/CD, deploys)
