name: change-budget
description: Ограничение масштаба изменений: маленькие диффы, одна ось изменений.
---

Для любых изменений действует бюджет правок.

Базовые ограничения:
- Предпочтительно ≤ 8 затронутых файлов за итерацию.
- Только ОДНА ось изменений:
  - либо рефакторинг,
  - либо обновление зависимостей,
  - либо изменение поведения.
- Не смешивать оси в одной итерации.

Если задача превышает бюджет:
1) Разбить на фазы.
2) Завершить фазу полностью (проверки + фиксация).
3) Только потом переходить к следующей.

Особые случаи:
- “Сделать компактнее / убрать легаси / обновить всё” — всегда начинать с фазного плана.
- Крупные правки требуют регрессионных якорей до упрощения кода.

Ограничения:
- Skill не выполняет рефакторинг сам.
- Skill не заменяет архитектурные решения.
- Цель — управляемость и предсказуемость, а не скорость.
