## Обзор

Собирай знания о проекте через интервью и поддерживай минимальный набор docs.
Только документация — код не меняй.

## Целевые документы

| Файл | Содержание |
|------|------------|
| `docs/architecture.md` | Модули, потоки данных, точки расширения |
| `docs/tech-stack.md` | Языки, фреймворки, инфраструктура |
| `docs/runbook.md` | Запуск, тесты, env vars, типичные ошибки |
| `README.md` | Краткое описание + быстрый старт |

## Workflow

1. Проверь наличие файлов
2. Читай существующее, дополняй аккуратно
3. Задавай вопросы по группам: архитектура → стек → запуск
4. Обновляй документы по ходу ответов
5. Если информации нет — ставь маркер "Нужно уточнить: ..."

## Вопросы для интервью

**Архитектура**: модули и их роли, потоки данных, точки расширения
**Стек**: языки/версии, ключевые зависимости, хранилища
**Запуск**: команды, env vars, типичные ошибки

## Правила

- Только документация — код не трогать
- Кратко, списками, команды в код-блоках
- UTF-8
