## Обзор

Собирай требования через интервью, предлагай два варианта структуры.
Создавай только каркас — без бизнес-логики.

## Workflow

1. Интервью: цель, входы/выходы, окружение, зависимости
2. Предложи два варианта с trade-offs
3. Дождись выбора пользователя
4. Покажи план файлов перед созданием

## Варианты структуры

**A. Минимальный** — для скриптов и узких задач:
```
src/<pkg>/main.py
tests/test_smoke.py
pyproject.toml, README.md
```

**B. Масштабируемый** — для долгоживущих проектов:
```
src/<pkg>/
  core/config.py, logging.py
  adapters/, services/, cli/
tests/unit/, integration/
pyproject.toml, Makefile, .env.example
```

## Что создавать

- `pyproject.toml` — ruff, pytest, mypy
- `Makefile` — lint, test, run
- `core/config.py` — загрузка конфигурации
- `core/logging.py` — базовое логирование
- Пустые `__init__.py` для структуры

## Правила

- Только каркас, без бизнес-логики
- Явные зависимости и интерфейсы
- Строгая типизация
